# Employee Workflow Tracking Project Context

## Current Delivery Priorities

1. Frontend approval is the first milestone and should be ready within one day.
2. Full project submission is due in three days.
3. The PDF requirements are more important than matching the ER diagram or database flowchart exactly.
4. The user is a beginner intern and needs explanations while building, not only final code.

## Agreed Build Strategy

1. Build a polished frontend demo first.
2. Confirm requirements from the PDF.
3. Add backend APIs to support the approved frontend.
4. Finalize database schema and seed data.
5. Prepare manager-facing submission notes and run instructions.

## Frontend Approval Scope

- Admin dashboard
- Employee directory
- Attendance overview
- Real-time activity tracking view
- Productivity analytics
- Reports view
- Settings/security indicators

## Requirement Source Summary

The PDF requires employee login/logout tracking, idle time detection, productive hour calculation, attendance reports, admin-level monitoring, employee dashboards, real-time widgets, reports/exports, notifications, secure authentication, MySQL-backed data, REST APIs, documentation, and a polished responsive animated frontend.
